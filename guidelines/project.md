# Project Context

## Business Type
This project is a web application for machine learning and AI challenges.
The product is conceptually similar to LeetCode, but focused on practical ML work instead of algorithm interviews.

## Product Goal
The platform helps users practice real machine learning tasks through structured challenges, submissions, and scoring.
It is designed for learners and junior practitioners who want hands-on repetition with realistic ML workflows.

## Core Experience
- Users browse challenge sets by topic or difficulty.
- Users complete ML tasks such as classification, regression, feature engineering, model evaluation, and error analysis.
- Users submit predictions, outputs, or other deliverables depending on the challenge.
- The platform evaluates submissions using clear metrics such as Accuracy, Precision, Recall, MSE, and RMSE.
- Users improve through repeated attempts, score comparison, and progressive difficulty.

## Product Positioning
- "LeetCode for machine learning and AI" is an acceptable shorthand for the product.
- The focus is practice and applied skill-building, not theory-heavy coursework.
- The experience should feel like solving realistic ML tasks, not just reading lessons.

## Implementation Guidance
- New features, copy, UI, and architecture decisions should align with the ML challenge platform direction.
- Avoid introducing business language, entities, or flows from unrelated domains such as real estate unless explicitly requested.
- If older code or assets reflect a previous domain, treat them as legacy or transitional unless the user says otherwise.
